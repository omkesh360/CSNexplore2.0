===========================================================
    HOW TO UPLOAD YOUR WEBSITE TO HOSTINGER
===========================================================

Your website is currently configured to use an SQLite database (`database/csnexplore.db`). This is great because it means everything is portable! You do not actually need to use the MySQL database you created on Hostinger.

Here is the easiest way to get your website online:

-----------------------------------------------------------
OPTION 1: The Easiest Way (Using your existing SQLite DB)
-----------------------------------------------------------

This method takes 5 minutes and works perfectly out-of-the-box.

1. Zip your website files:
   - Go to your project folder (`c:\xampp\htdocs\CSNexplore2.0`)
   - Select all files and folders (including the `database` folder!).
   - Right-click and compress them into a single `.zip` file.
   
2. Upload to Hostinger:
   - Log into your Hostinger hPanel.
   - Go to Websites -> Manage -> File Manager.
   - Open the `public_html` folder.
   - Upload the `.zip` file you created.
   - Right-click the `.zip` file and select "Extract".

3. Set Permissions (Important):
   - Right-click the `database` folder and its file (`csnexplore.db`) inside Hostinger's File Manager.
   - Set the permissions to 775 (or 777 if 775 doesn't work) so that PHP can read and write to the database file.

That's it! Your website will now be live with all your existing data.

-----------------------------------------------------------
OPTION 2: The Advanced Way (Using the Hostinger MySQL Database)
-----------------------------------------------------------

If you absolutely must use the MySQL database you already created on Hostinger, you will need to change the code, because SQLite and MySQL have different SQL syntax.

To do this, you would need to:
1. Open `php/config.php` and add your MySQL database credentials (Username, Password, DB Name, Host).
2. Rewrite `php/database.php` completely. You have to change `PDO('sqlite:...')` to `PDO('mysql:host=...')`.
3. Rewrite the `CREATE TABLE` queries inside `database.php` because MySQL uses `INT AUTO_INCREMENT PRIMARY KEY` instead of SQLite's `INTEGER PRIMARY KEY AUTOINCREMENT`.
4. Migrate all your existing data from SQLite to MySQL.

**Recommendation:** Stick to Option 1! Option 1 requires zero configuration and will keep all the images and data you've already added.
