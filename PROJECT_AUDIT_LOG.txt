SECTION A — TECHNICAL GAPS (Code & Infrastructure)
================================================================================

──────────────────────────────────────────────────────────────────────────────
A1. SECURITY (Most Urgent)
──────────────────────────────────────────────────────────────────────────────

★ [A1.1] CSRF Protection is MISSING
   - Status: Booking form, contact form, subscribe form have NO CSRF tokens.
   - Risk: Attackers can forge form submissions on behalf of logged-in users.
   - Fix: Generate a token per session, embed in forms, validate on POST.

★ [A1.2] SMTP Credentials are NOT configured
   - Status: config.php has SMTP_USERNAME = '' and SMTP_PASSWORD = ''
   - The EmailService exists and is well-written, but email is DEAD because
     SMTP credentials are blank. No booking confirmation emails are sending.
   - Fix: Add Gmail App Password or real SMTP credentials in the .env file.

  [A1.3] SSL Verification is Disabled in PHPMailer
   - Status: EmailService.php has 'verify_peer' => false — this is a dev hack.
   - Risk: Man-in-the-middle attacks on email sending in production.
   - Fix: Remove the SMTPOptions block when deploying to production.

★ [A1.4] JWT Secret is Hardcoded as Default
   - Status: JWT_SECRET defaults to 'csnexplore_secure_jwt_2025_!@#$%' in code.
   - Risk: If repo is public or leaked, all admin tokens can be forged.
   - Fix: Move it exclusively to .env, remove default fallback.

  [A1.5] Admin API endpoints lack Rate Limiting
   - Status: /php/api/auth.php (login) has no brute-force protection.
   - Risk: Password guessing attacks possible on admin login.
   - Fix: Add a simple IP-based rate limiter (max 5 attempts per minute).

  [A1.6] File Upload API is Basic
   - Status: /php/api/upload.php accepts uploads with minimal validation.
   - Risk: Malicious file uploads (e.g., PHP files disguised as images).
   - Fix: Validate MIME type (not just extension), restrict to image types only.

──────────────────────────────────────────────────────────────────────────────
A2. AUTOMATION & WORKFLOW GAPS
──────────────────────────────────────────────────────────────────────────────

★ [A2.1] Blog/Listing Save does NOT auto-regenerate static HTML
   - Status: When admin saves a blog in blog-editor.php, the PHP API saves to DB.
     But the static HTML file in /blogs/ is NOT updated automatically.
   - Impact: The live website shows OLD content until someone manually runs
     generate_html.php via command line.
   - Fix: Call the generate_html function at the end of the PUT/POST handler
     in /php/api/blogs.php and /php/api/listings.php for that specific item.

  [A2.2] No Database Backup System
   - Status: No automated MySQL dump or backup schedule exists.
   - Risk: One MySQL crash = 300 blogs, 74 listings, all bookings GONE forever.
   - Fix: Create a backup.php script and schedule it via Windows Task Scheduler
     or a Cron Job on the live server. Email the backup to admin daily.

  [A2.3] Sitemap Generation is Manual
   - Status: /php/api/generate_sitemap.php exists but must be manually visited.
   - Fix: Auto-trigger sitemap regeneration after any new blog/listing is added.

  [A2.4] Cache Invalidation is Missing
   - Status: Performance cache is built, but when a blog is updated, the OLD
     cached version still gets served until the cache expires.
   - Fix: On content save, delete the specific cached HTML file for that page.

──────────────────────────────────────────────────────────────────────────────
A3. MISSING FEATURES (Technical)
──────────────────────────────────────────────────────────────────────────────

  [A3.1] "Welcome" Email on Registration is Missing
   - Status: auth.php creates user accounts but never sends a welcome email.
   - EmailService class EXISTS and has PHPMailer wired up — just not called.
   - Fix: After INSERT into users, call EmailService::sendWelcomeEmail().
     (This method needs to be written in EmailService.php + template added.)

  [A3.2] User Reviews / Ratings System is Missing
   - Status: 'rating' and 'reviews' are just flat numbers in each table.
     There is NO separate reviews table, NO way for users to submit reviews.
   - Fix: Create a 'reviews' table (user_id, listing_type, listing_id, rating,
     comment, created_at). Add a review form on listing detail pages.
     Average ratings dynamically from this table.

  [A3.3] Predictive / Auto-complete Search is Missing
   - Status: Search on index.php and listing.php is a simple form submit.
   - Impact: Users type "Ajant…" and get nothing until they press Enter.
   - Fix: Add a debounced fetch to /php/api/listings.php?search=... and
     render a dropdown with matching results.

  [A3.4] No Pagination on Blogs Page
   - Status: blogs.php loads ALL 300 blogs at once from the database.
   - Impact: Extremely slow page load. Bad for SEO (duplicate content risk).
   - Fix: Add LIMIT/OFFSET in the SQL, add page number links in the UI.

  [A3.5] No Pagination on Listing Page
   - Status: listing.php also loads ALL items at once.
   - Fix: Same LIMIT/OFFSET approach + AJAX "Load More" button.

  [A3.6] No "Forgot Password" Email Flow
   - Status: forgot-password.php and reset-password.php pages exist in the
     file listing but the actual email sending may not be wired up properly
     (SMTP is blank — see A1.2).
   - Fix: Configure SMTP first, then test end to end.

  [A3.7] Vendor Dashboard is Missing
   - Status: The users table has role='vendor' defined but there is NO
     vendor-specific dashboard. Vendors cannot log in and manage listings.
   - Fix: Create /vendor/ directory with pages for managing only their items.

  [A3.8] My Bookings page needs enhancement
   - Status: my-booking.php exists (track booking). A full "My Account" area
     with booking history, profile editing, etc., is missing.
   - Fix: Build a proper user dashboard at /my-account.php.

  [A3.9] No Image Upload from Admin UI for Listings
   - Status: upload.php API exists. But in admin/listings.php, the image field
     appears to just be a URL text input, not a file upload button.
   - Fix: Wire the upload.php API to an image upload button in listings editor.

  [A3.10] Blog Editor has no Image Upload
   - Status: blog-editor.php uses a plain URL input for the featured image.
   - Fix: Add a "Upload Image" button that calls /php/api/upload.php and
     pastes the resulting URL into the field automatically.

  [A3.11] No Social Login (Google / Facebook)
   - Status: Only email/password login exists.
   - Fix: Integrate Google OAuth 2.0 for one-click sign-in. Very expected
     by modern users on travel websites.

──────────────────────────────────────────────────────────────────────────────
A4. SEO TECHNICAL GAPS
──────────────────────────────────────────────────────────────────────────────

  [A4.1] Dynamic pages (listing.php, blogs.php) have Generic Meta Tags
   - Status: These PHP pages don't pull unique <title> or <meta description>
     from the DB for individual items.
   - Fix: Pass a $pageMeta array into header.php from each page.

  [A4.2] No Pagination Canonical Tags
   - Status: No rel="prev" / rel="next" link tags when listing is paginated.
   - Fix: Add these in header.php based on current page number.

  [A4.3] Hotel/Restaurant Schema Markup is Wrong
   - Status: generate_html.php uses 'LocalBusiness' schema for hotels.
     should use 'Hotel', 'LodgingBusiness', or 'FoodEstablishment' schema
     for richer Google results (Star ratings, price shown in search).
   - Fix: Update the $schemaType logic in generate_html.php per listing type.

  [A4.4] Images have no alt text strategy
   - Status: onerror fallbacks exist, but alt="" tags are often generic.
   - Fix: Use the listing name dynamically in alt tags.

  [A4.5] No 301 Redirect for Old URLs
   - Status: If listing slugs change (e.g., admin renames a hotel), old
     static HTML files become dead pages with no redirect.
   - Fix: Track old slugs in DB, auto-add 301 rules to .htaccess or a table.
